# aegis-proxy

Cloudflare Worker that fronts Anthropic for the peeky desktop client.

## What it does

Holds the secret Anthropic, Deepgram, and Cartesia API keys server-side so the
desktop binary can ship without users needing to sign up and paste keys. Caps
usage from a KV store. Streams Anthropic responses; mints short-lived tokens
for Deepgram and Cartesia (the client opens its own WebSocket to those).

## Tiers

Selected per request, no account required.

- **trial** (default, no header): lifetime call counter, capped by
  `TRIAL_TURNS_CAP`. Default 9 = 3 voice queries at 3 calls each.
  Soft-resets after 30 days of inactivity (KV TTL).
- **demo** (request carries `x-peeky-invite-code`): the same lifetime counter
  with a higher cap (`turns_cap`) from the code's KV payload. Used for
  recruiter demos and hand-granted access. Mint with `scripts/mint-code.sh`.

## Endpoint

```
POST https://aegis-proxy.<your-subdomain>.workers.dev/v1/anthropic/messages

Headers:
  content-type: application/json
  anthropic-version: 2023-06-01
  anthropic-beta: <whatever the client sends>
  x-peeky-device-id: <uuid generated by the client on first launch>
  x-peeky-invite-code: <optional; promotes the request to the demo tier>

Body:
  Standard Anthropic Messages API request. `stream: true` is required.
```

Response is identical to Anthropic's, including SSE streaming, with one
exception: requests past the device's lifetime cap return `429` with a JSON
body (`trial_exhausted` or `code_exhausted`).

## Deploy

You need a Cloudflare account (free). The rest takes ~10 minutes.

### 1. Install the wrangler CLI

```bash
npm install
```

This pulls `wrangler`, `typescript`, and Cloudflare's type definitions into
`node_modules/`. Nothing global is installed.

### 2. Log in to Cloudflare

```bash
npx wrangler login
```

Opens a browser tab, authorizes wrangler against your account.

### 3. Create the KV namespace

```bash
npx wrangler kv namespace create USAGE_KV
```

Output looks like:

```
🌀 Creating namespace with title "aegis-proxy-USAGE_KV"
✨ Success!
[[kv_namespaces]]
binding = "USAGE_KV"
id = "abcd1234..."
```

Copy that `id` into `wrangler.toml`, replacing `REPLACE_WITH_KV_NAMESPACE_ID`.

### 4. Set the Anthropic key as a secret

```bash
npx wrangler secret put ANTHROPIC_API_KEY
```

Paste your `sk-ant-...` key when prompted. It's stored encrypted and is never
visible afterward, even to you.

### 5. Deploy

```bash
npx wrangler deploy
```

Output gives you the worker URL, e.g.
`https://aegis-proxy.your-subdomain.workers.dev`. That's the base URL the
peeky client points at.

### 6. Sanity check

```bash
curl -X POST https://aegis-proxy.your-subdomain.workers.dev/v1/anthropic/messages \
  -H "content-type: application/json" \
  -H "x-peeky-device-id: $(uuidgen)" \
  -d '{
    "model": "claude-haiku-4-5",
    "max_tokens": 64,
    "stream": true,
    "messages": [{"role": "user", "content": "say hi"}]
  }'
```

You should see SSE events stream back.

## Usage caps

Both tiers meter the same way: a lifetime count of calls per device. Each
voice query is about 3 calls (STT token, Claude, TTS token). The trial cap is
`TRIAL_TURNS_CAP` in `wrangler.toml` (default 9, so 3 voice queries). Invite
codes carry their own `turns_cap`. Counters soft-reset after 30 days of
inactivity. Tune the trial cap in `wrangler.toml` or a code's payload, then
redeploy with `wrangler deploy` to apply.

## Local development

```bash
npm run dev
```

`wrangler dev` runs the Worker locally with a fake KV namespace, against the
real Anthropic API (using the secret you've set). Hot reloads on save.

To use a local Anthropic key during dev without setting a real secret, create
`.dev.vars` (gitignored):

```
ANTHROPIC_API_KEY=sk-ant-...
```

## Architecture (one paragraph)

Worker receives a POST → reads `x-peeky-device-id` → resolves the tier (trial,
or demo when a valid invite code is present) → charges one call against the
device's lifetime counter in KV (`usage:trial:{deviceId}` or
`usage:demo:{code}:{deviceId}`) → 429s if over the cap → otherwise forwards the
body to Anthropic with our secret key and streams the SSE response straight
through. The Deepgram and Cartesia token-mint endpoints charge a call the same
way before minting. Counters carry a 30-day TTL, refreshed on each use.

## Invite codes

Mint a code for a specific recipient (recruiter, beta tester, etc.) and hand
it to them out-of-band. The recipient pastes it into the Peeky client, which
sends it as `x-peeky-invite-code`.

```bash
./scripts/mint-code.sh RECRUITER-ACME 50 2
# Code:        RECRUITER-ACME-A3F1B2
# Uses:        50 (150 calls)
# Max devices: 2
```

Args are `<label> [uses] [max_devices]`. Codes do not expire; they run until
their lifetime uses are spent.

Codes are stored in `USAGE_KV` as `invite:{CODE}`. Edit caps by either
re-minting (overwrites the old key with the same code, if you reuse the
suffix) or `wrangler kv key put` directly. Delete a code with
`wrangler kv key delete --binding=USAGE_KV invite:{CODE}`.
